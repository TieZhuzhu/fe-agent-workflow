# UI 截图分析

对用户提供的界面截图（设计稿、PRD 原型图、Figma 导出图等）做结构化分析，为后续技术方案与代码生成提供输入。

输出格式与 [prd-feature-dev](../prd-feature-dev/SKILL.md) 的「需求摘要」对齐，便于直接进入方案对齐阶段。

---

## 分析维度

### 1. 终端识别

- **小程序 / App**：底部 Tab、手机框、竖屏、圆角卡片（**本项目默认目标**）
- **PC 后台**：宽屏、侧栏 + 表格 — 须在 digest 标注，实现时**转换为移动端列表**
- **PDA**：简化移动端

### 2. 布局结构

- 页面整体结构：页头、筛选区、主内容、底栏
- 区块嵌套关系与对齐方式
- **复杂页识别**：是否存在多个独立区块（统计卡片 + 列表 + 侧栏等）

### 2b. 视觉还原范围（Figma / 设计稿截图）

分析时区分**内容区**与**系统 chrome**：

| 纳入还原（写入样式 token） | 排除（不写入代码） |
|---------------------------|-------------------|
| 业务顶栏标题、返回、右侧按钮 | 系统状态栏（时间/电量/信号） |
| 列表/卡片/表单/按钮样式 | 微信小程序胶囊 |
| 间距、字号、颜色、圆角 | 底部 Home 指示条 |
| 固定底栏业务操作区 | 手机外壳、设备边框 |

输出须含 **样式 token 表**（主色、标题字号、页边距等 rpx 值），供 [figma-feature-dev](../figma-feature-dev/SKILL.md) 百分百还原。

### 3. 组件识别

**操作类**：主按钮（填充）、次按钮（描边）、文字按钮（无边框）

**表单类**：输入框、下拉、日期（单日期 vs 日期范围：单框 vs 两框连字符）、级联、单选/多选

**展示类**：列表项（卡片/行）、描述列表、统计卡片（PC 表格须转为列表项展示）

**反馈类**：弹窗、抽屉、确认框、loading、空状态

### 4. 交互行为

- 点击、输入、筛选、分页、批量操作
- 弹窗打开/关闭时机
- 页面跳转或 Tab 切换
- 权限相关交互差异

### 5. 内容与状态

- 表格列名、表单 label、按钮文案
- 正常 / 空数据 / loading / 错误 / 禁用 / 选中 状态

### 6. 多图关系

多张图时分析：流程先后、主从关系、分支流程

## 冲突处理

截图内容与用户提供的 PRD/接口文档冲突时：**以 PRD/接口文档为准**。

- 截图列名 → 仅作 label
- 截图中的示例数据 → 不作 prop 名、不作 mock
- 截图有但 PRD/接口未提及的功能 → 列入待确认
