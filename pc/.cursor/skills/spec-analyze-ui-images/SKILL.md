# UI 截图分析

> **管控力度：** 松 | **规范：** [skill-conventions](../shared/skill-conventions.md) | **边界：** [baseline](../shared/skill-boundaries-baseline.md) | **工具：** [toolbox](../shared/project-toolbox.md)

对用户提供的界面截图（设计稿、PRD 原型图、Figma 导出图等）做结构化分析，为后续技术方案与代码生成提供输入。

输出格式与 [prd-feature-dev](../prd-feature-dev/SKILL.md) 的「需求摘要」对齐，便于直接进入方案对齐阶段。

---

## 分析维度

### 1. 终端识别

- **PC**：宽屏、横向布局、侧边栏 + 主内容区
- **App**：移动端特征，底部导航、圆角
- **PDA**：简化移动端，无装饰性圆角

### 2. 布局结构

- 页面整体结构：页头、筛选区、主内容、底栏
- 区块嵌套关系与对齐方式
- **复杂页识别**：是否存在多个独立区块（统计卡片 + 列表 + 侧栏等）

### 3. 组件识别

**操作类**：主按钮（填充）、次按钮（描边）、文字按钮（无边框）

**表单类**：输入框、下拉、日期（单日期 vs 日期范围：单框 vs 两框连字符）、级联、单选/多选

**展示类**：表格（表头、列、操作列）、卡片、列表、描述列表

**反馈类**：弹窗、抽屉、确认框、loading、空状态

### 4. 交互行为

- 点击、输入、筛选、分页、批量操作
- 弹窗打开/关闭时机
- 页面跳转或 Tab 切换
- 权限相关交互差异

### 5. 内容与状态

- 表格列名、表单 label、按钮文案
- 正常 / 空数据 / loading / 错误 / 禁用 / 选中 状态

### 6. 多图关系

多张图时分析：流程先后、主从关系、分支流程

## 冲突处理

截图内容与用户提供的 PRD/接口文档冲突时：**以 PRD/接口文档为准**。

- 截图列名 → 仅作 label
- 截图中的示例数据 → 不作 prop 名、不作 mock
- 截图有但 PRD/接口未提及的功能 → 列入待确认
