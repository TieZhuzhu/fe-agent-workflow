# Spec: 全项目代码规范化重构

> Feature slug: **project-refactor**

## 用户故事

- 作为**前端维护者**，我希望存量代码符合统一规范，降低协作与回归成本。
- 作为**业务用户**，我希望重构后功能与界面行为与现网一致。

## 功能需求（重构类 FR）

| ID | 需求 | 优先级 |
|----|------|--------|
| FR-1 | 清理 dead code、mock、业务 console.log | P0 |
| FR-2 | 列表页统一 HiTable + 标准布局（接口允许时） | P0 |
| FR-3 | 页面目录 kebab-case + index.vue | P1 |
| FR-4 | 接口页面级 services，入参透传、出参直绑 | P1 |
| FR-5 | 全项目 lint + build 通过 | P0 |
| FR-6 | 分阶段模块冒烟无功能退化 | P0 |

## 验收场景（业务回归 — 必填）

### 场景 1：登录后主流程

- **Given** 有效账号
- **When** 登录后依次进入（填写本项目核心模块 path）
- **Then** 页面正常加载，数据与重构前一致，无控制台报错

### 场景 2：列表分页与筛选

- **Given** 任一核心列表页有数据
- **When** 切换页码、修改筛选并查询
- **Then** 列表与 total 正确，loading 正常

### 场景 3：构建与静态检查

- **Given** 当前分支全部重构改动
- **When** 执行 `npm run lint-fix` 与 `npm run build`
- **Then** 零 ESLint error，构建成功

### 场景 4：空态 / 错误

- **Given** 筛选无匹配数据
- **Then** 展示空态，不抛未捕获异常

> 按项目补充：订单详情、权限、导出等 P0 路径。详见 `e2e.md`。

## 非功能

- **行为不变**：路由 path、接口 URL、文案不变
- **最小 diff**：单模块单 PR
- **禁止 mock**：清理遗留 mock，不新增

（与 design.md、inventory 一致，P0 基础设施 → P1 核心交易 → P2 设置 → P3 其余）
